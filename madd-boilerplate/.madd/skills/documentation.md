# Skill: Documentation

> Skill for writing clear, useful documentation.

## Description

This skill enables an agent to create and maintain documentation that helps users and developers understand and use the project.

## Capabilities

- Write user-facing documentation
- Document APIs and interfaces
- Create tutorials and guides
- Maintain README files
- Generate inline documentation

## Documentation Types

### User Documentation
- Getting started guides
- Feature explanations
- Configuration reference
- Troubleshooting guides

### Developer Documentation
- Architecture overview
- API reference
- Contributing guide
- Development setup

### Inline Documentation
- Function/method documentation
- Complex logic explanations
- TODO/FIXME annotations

## Writing Guidelines

### Clarity
- Use simple, direct language
- One idea per sentence
- Define jargon on first use
- Use active voice

### Structure
- Start with the most important information
- Use headings to organize
- Include examples
- Provide quick-start for impatient readers

### Maintenance
- Keep docs next to code when possible
- Update docs with code changes
- Remove outdated information
- Version documentation with releases

## Documentation Template

```markdown
# Feature Name

Brief description of what this does.

## Quick Start

Minimal example to get started.

## Usage

Detailed usage instructions.

## Configuration

Available options and settings.

## Examples

Real-world usage examples.

## Troubleshooting

Common issues and solutions.
```

## Anti-patterns

- Documentation that restates the code
- Outdated documentation (worse than none)
- Wall of text without structure
- Assuming too much prior knowledge
